AImage uniformity

Bands are uniform; sizes come in exactly two grids.

Every readable file is 4-band uint8 in EPSG:3857 at ~4.78 m/px — so “all RGB” is right except for a 4th alpha / validity mask band you'll want to drop or use for filtering. Pixel dimensions are not all equal: 2017 Q1&Q2 were delivered as half-size 2048 px tiles (same resolution, smaller footprint), everything else is 4096 px.

BSpatial coverage

Clusters line up with the crosswalks; most straddle more than one quad.

All 19,241 clusters across the eight downloaded years match the crosswalk_quads_planet_*.geojson files. Multi-quad clusters (a cluster's buffer spanning 2+ quads) are expected and harmless — the square buffer either sits inside one quad, straddles a grid line (2) or a corner (4). 2017 is different: its 2048 px tiles are half-size, so every 2017 cluster touches 5–13 quads.

512 divides both grids exactly (8×8 and 4×4), so no remainder. Two adjustments change this number: filtering patches by the alpha band (many quads are <100% covered) drops it, and de-duplicating the three quarters / repeated tiles across years (for spatially non-overlapping output) divides it by roughly three.

⚡

Fastest extraction: the bottleneck is reading ~3.5 TB off the mount, so — parallelise across ~48 worker processes, read each quad once (block-aligned), drop empty patches via alpha, and write a few thousand patches per shard to local scratch rather than millions of tiny files on the network FS.
